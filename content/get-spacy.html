Dependencies

Spacy itself does not require anything than a recent C++-compiler with support for C++14. For testing googletest is used and will be downloaded by cmake.

The interfaces in Spacy are automatically generated using a custom clang tool.

Installation

Spacy can be built with cmake. The usual installation procedure is:

  • cd spacy/root/dir
  • mkdir build
  • cd build
  • cmake ..
  • make

Libraries that are build with cmake, such as Eigen or FEniCS, should be detected by cmake. For usage with Kaskade7 run cmake -DKaskade=ON ... For building the tests you have to specify cmake -DBuildTest=ON ...

Contributing

Using Gerrithub

In order to contribute to Spacy you need to get the project through gerrithub.io:

  • Get an account at github.com.
  • Use this account to sign in at gerrithub.io.
  • Go to Projects->List and find Spacy (using the filter).
  • Choose 'Clone with commit-msg hook' and one of anonymous http, http or ssh and copy the shown command (git clone ...) to clipboard.
  • Execute this command in terminal from the folder in which you want to place the Spacy folder.
  • Check if the email-address of git matches the one registered at github/gerrithub. To retrieve the address that is locally registered in git execute git config --get user.email. If this address does not match the one registered at gerrithub, then you need to adjust it via git config (--local or --global) user.email "email@example.com".

Backlog

Spacy currently uses waffle.io to track tasks and bug-fixes. If you are running out of ideas and want to contribute, then check the backlog for open tasks.

Formatting

Formatting rules are specified in .clang-format in Spacy's root folder. To automatically format your changes install clang-format and copy the pre-commit from hooks/pre-commit to .git/hooks/.

Ubuntu:

  • sudo apt-get install clang-format
  • cd spacy/root/dir
  • cp hooks/pre-commit .git/hooks/
